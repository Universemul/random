P. 3. Paraphrase of Definitions of words, action items, and ordering

The following is a different wording of the same specification; it might be more clear than the original.


1. A word is between 2 and 31 characters long, delimited by whitespace, punctuation, or the beginning/end of a line.


2. A word can contain only alphabetic characters except that a word can contain any number of apostrophes so long as they are not at the beginning or end of the word, in which case they are treated as punctuation.


3. The possessive form "'s", when found at the end of a word, is removed before any operations are performed, but it is expected to be preserved in the output of any spell-checked document. A word must be at least 2 characters long after removal of "'s". Recursive removal is not expected.  "Tom's's" becomes "Tom's" rather than "Tom".


4.  It is intended that the apostrophe be removed from the plural possessive as well, but this needs no special provision due to the basic definition of a word as defined in points #1 and #2.


5. Words typed in different case (upper/lower) must be recognized as the same word in the dictionary. In this project, the words in the dictionary must be all lower-case, so you must convert words to lower case for use in the dictionary and to compare against the dictionary. The output document, however, should preserve the original case of correctly spelled words. For example,  the program checks the spelling of "David Kieras" as if it were "david kieras" but if correct, the output document contains "David Kieras".


6. The words in the action list must be kept in the exact case, which means that the program allows more than one action to be specified for the same word if they have different cases. It is possible to both skip the word "Pig" and replace "PIG" with "DOG".


7.To avoid confusion as to what is meant by alphabetic characters, punctuation, or whitespace, see the handout on <ctype.h> C Library functions for character types on the course website. 


a. Alphabetic characters are the letters A through Z, either upper or lower case, as defined by the <ctype.h> function isalpha. 


b. The whitespace characters are defined by the function isspace.


c. The punctuation characters are defined by ispunct. 


Thus, words cannot contain numbers, whitespace, or characters like '!', and to be considered a word, a string of characters must be preceded and followed by a whitespace, punctuation, or the beginning or end of a line.


6.Throughout this document, alphabetical order means in the usual alphabetical order and as defined by the strcmp function in <string.h> (see the handout C Library C-string functions on the course website). That is, if strcmp(s1, s2) returns a negative value, it means that s1 comes before s2 in the alphabetical order.



P. 3, After Item #7. Word processing goes one word at a time. 

Add:

7a. The program processes the input document one word at a time; the program finds each word, checks it against the dictionary, and takes any action or queries the user, before the program goes to the next word. 


P. 9, Programming Requirements, Specific concepts and techniques. Restriction on use of memory allocation functions.

After Item #11, add:

12. The only Standard C Library function you can use to allocate memory in this project is malloc. Other Library functions such as calloc or realloc may not be used. Also, you may not use non-Standard library functions such as strdup that allocate memory. You must use only malloc and code of your own instead.

Explanation:

The other Standard C Library functions such as calloc and realloc are occasionally useful, but have more complex interfaces, hide some important details, and do very little more than you can do with your own code and malloc. malloc is the simplest and most basic memory allocation function, and so you will understand the concepts better by using it to allocate memory.


P. 9, 10. Restriction on global variables does not apply to file-scope global constants defined as const variables.

For example, you can define the message strings from strings.txt with either #define symbols or as initial values for const char * const variables at file scope in the relevant .c file. Global const variables, since they are unmodifiable once initialized, don't have the debugging and maintenance problems of regular global variables because they can't be used as a communication channel between functions.


Conflict between Action_item enum type name and C Coding Standards.

According to the C Coding Standards, the enum should be terminated with "_e" instead of "_t". The C Coding Standards are a recent addition to the course materials, and I failed to review the project specifications in this regard.


The Action_item.h header file was updated to use "_e" for this name as of 10:30 PM Jan. 17. If you have already written this module, use global find&replace to change the name to match; if not, be sure to download the updated version of the header file.
