I'm having trouble understanding the definition of a word and exactly how the spell-checking is supposed to work.

Believe it or not, a real effort has been made to keep the project as simple as possible, given what it is supposed to do. So often times, if it isn't clear what is supposed to happen, the simplest possibility is the right answer.


A good approach is to first, see the above question, and review the program's functionality (starting bottom of page 2) and study the console samples to get the big picture of how the program is supposed to behave.


Then ask yourself, "What do I need to know in order to write the code?" Lots of little nit-picky things, right? Such as, just what is a word? Try sketching what your code will do and then look to the specs to answer the specific questions as they come up.


The apostrophes are a tricky business. If you find it confusing, work out how you would identify words for spell-checking without any apostrophe's involved. Then see how you would need to modify things to take them into account.

Can I use the Standard C Library bsearch function to search the Ordered_array?

It is permitted, but it has two disadvantages:


1. Because bsearch is as generic as possible, it does not assume anything about what the array elements consist of. It does the arithmetic using the supplied parameters to figure out the address of each cell of the array, and refers to these cells with void pointers to the cells. So the comparison function is not called with elements of the array (which would be void *'s for Ordered_array), but rather with void pointers to the elements (void **'s in the case of Ordered_array). See the course website "Code Examples/C_examples" for a couple of simple examples of using bsearch to see how to set it up. The problem is that the comparison function as specified in the project documents will not work directly with bsearch.


2. The bsearch function will tell you whether an item is in the array, and if so, where it is. But if it is not in the array, bsearch won't tell you where to put it, which of course is something that you will figure out for Ordered_array.


These disadvantages mean that writing your own binary search function would be the better choice - see K&R p. 58 for a good version to use as a starter. Note that in this version, if the item is not found, the subscript high + 1 is where it should be inserted.

What compiler, version, and options are being used by the autograder?

gcc 4.1.1, with compile options of


-ansi -pedantic -std=c89 -Wmissing-prototypes -Wall


Especially if you are using Windows or Mac OS X, before submitting your code, you should upload to a CAEN workstation and try compiling your code with gcc 4.1.1 or a later version with the same compiler options. Test with at least the samples and check for an exact match on output to help detect any platform-specific problems.


Use the supplied makefile, or do a quick-and-dirty build as follows: make sure the only files in the current directory are the ones you plan to submit. Enter:


gcc -ansi -pedantic -std=c89 -Wmissing-prototypes -Wall *.c


This will compile and link together all the source files in the directory and build an executable named a.out, which you can execute with:


./a.out
