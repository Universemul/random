My project is failing one of the tests that involves Views, but it is passing the samples and the other tests. Any suggestions?

The problem could be that your bridge_view is not correct. Reread the discussion on floating point issues in the Project 4 document, and take it extremely seriously. If you calculate the subscripts differently than specified, a tiny difference in the final double-float value can make the integer index value end up at a different bin. This can happen in a very random-looking way.


Reread the Project 5 spec doc on the specifications for bridge_view and look at your code to see if you do exactly the same calculations - not exactly in a mathematical sense, but in exactly the same sequence of operations. The spec refers to the get_subscripts function supplied for the map view - this should be the model for the last stage of this new code.
