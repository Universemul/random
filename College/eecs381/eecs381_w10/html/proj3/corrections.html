Your solution must follow these corrections and clarifications.


P. 2. Clarification of errors for the cc command. Error messages do not include collection name.

The listing of possible errors for cc mentioned the collection names only to help explain the order of the checking of the names - the names themselves do not appear in the error messages. The strings.txt files for this and the previous projects contain all of the error messages needed. See also the error and new commands samples.


P. 2. Errors for mt command. Failure to read an integer comes first.

Change:

Errors: No record with that ID; there is already a record with that title.

To:

Errors: Unable to read an integer; no record with that ID; there is already a record with that title.


P. 4. Exception #3, last bulleted item. Reading and condensing a title.

You must read the title into a std::string using std::getline, and then condense it without any explicit loops (last bullet). However, you can use an explicit loop to skip past whitespace or newlines to get to the start of the title before you do the getline (exception #3).
