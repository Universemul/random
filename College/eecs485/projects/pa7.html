Understanding AJAX

Database-driven web applications have been at the forefront of dynamic web content for a decade. The limitations of database-driven sites - static pages, dramatic reloads for even small changes to onscreen information - should have become apparent to you over the course of the semester so far. AJAX is designed to circumvent many of the technical limitations imposed by the web page based architecture in which we have been working.

Before you dive into writing code, take some time to familiarize yourself with how AJAX works. IBM has a pretty good introduction, but there are plenty of others on the Web.

Changing your UI.

Your first goal will be to improve your site's interface by creating a "gallery slideshow page." This page will show thumbnails of all the photos in an album, adjacent to a large "focus" image. The user should be able to select a thumbnail in order to change the content of the large focus image. This is roughly similar to the interface seen in Flickr. When the slideshow page is loaded, you should pick a default image to appear in the focus area. When the user clicks on a thumbnail image, the page should not reload - you should use AJAX to change the content of the focus. It is acceptable for this portion of the assignment to have the JavaScript be entirely client-side. That is, you do not need to have explicit server-side support. In other words, we have not yet seen the "X" part of AJAX. The goal here is just to become accustomed to using JavaScript.

Using Remote Data

In this portion of the assignment you will use AJAX to fetch data from a remote Web service and integrate it into your photo application. Add a feature to your site where the caption text of the currently-viewed photo is automatically sent as a query to (Amazon, Twitter, Flickr). Your JavaScript code should asynchronously send this information and get a reply from the service. The resulting (tweets, product-links, photos) should then be displayed alongside the photo that the user is currently viewing.

This service should work without much, if any, integration with your PHP-based server. Your code should contact the remote Web service only after the page is completely loaded by the user's browser.

The results should match the look and feel of your webpage, and not feel intrusive. This feature will be graded primarily on functionality, but with some attention devoted to how well the presentation meshes with your site.

Amazon Web Services

Using AJAX feauters, you can access Amazon's database via webservice. The next step will be to prepare to use the Amazon E-Commerce Service, which you will be using in this assignment. You will have to go to the Amazon Web Services homepage and register for an Access Key to use. Next, take some time to familiarize yourself with the operations of the E-Commerce API, whose Web Services Web Services Definition Language (WSDL) can be found here. WSDL is an XML format for describing network services as a set of endpoints operating on messages containing either document-oriented or procedure-oriented information. To learn more about WSDLs you can use the following resources:

  • http://www.w3.org/TR/wsdl
  • http://www.w3schools.com/wsdl/wsdl_intro.asp
For this assignment, you will be using the Amazon E-Commerce web service to provide links to books which may be relevant to the image someone is viewing. One presents keywords to the ItemSearch API, and receives book titles, authors, and links to Amazon pages in return. Here are a number of resources that you may find useful:
  • AWS documentations
  • Product Advertising API
  • An online tutorial on Amazon Search Servises
