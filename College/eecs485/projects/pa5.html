Assignment 5: XML import and export

Due Friday, February 26, 2010, at 11:55PM.

As described in class, one important use for XML is exchanging data between organizations. This is the situation where company A and company B each have a database with similar information, and they want to share this information. To do so they get together and choose a set of tags that describe the information they want to send to each other. Let's take our photo album scenario for example. You can imagine that users might want album information in places other than your site: on a desktop application, on a smartphone, on another album website, etc. This assignment will give you practice with packaging database information as XML messages (files) for outbound transfers and loading information into the database from incoming XML messages.

This assignment consists of two parts (i) exporting data from your database into XML and (ii) importing data from XML to your database. You will also use XSL to translate XML into HTML that can be rendered in a browser.

Part 1: What is XML?

Here are a set of resources which you may find useful:

  • XML Tutorial - Read about XML syntax, elements, attributes, validation, namespaces. No need to read about JavaScript or other programming issues.
  • W3C XML Stylesheet Language Spec and A Simple XSL Tutorial.
  • XML Parser Functions in PHP (Event-driven XML Parser).

Part 2: Export data to XML

On your album editing page, add a new link for each album. This link is used to display an XML page containing all the information about this album. By "display" we mean that you should print this XML page to the screen as you would any other page from a PHP script. (You can find a sample XML file here.) If you have correctly extracted the data and your XML is well-formed, a modern browser will render it in a tree.

Second, add a link that allows users to load a new album from an XML file, as discussed in Part 4.

Part 3: Make it pretty

Download this sample XSLT stylesheet file. Create your own XSLT file based on the one provided. Experiment with different styles; come up with something interesting and that fits in with the theme of your site.

Modify your album editing page such that the user can choose whether the XML page is displayed with the stylesheet applied or not. To apply the style sheet, include a reference to the XSL file in your XML output: 

<?xml-stylesheet type="text/xsl" href="pa5.xsl"?>

Part 4: Import data from XML

When the user clicks on the link to load a new album, he or she should be presented with a form. This form has just one text field used to specify a URL for an XML file containing all the information about the album to load. You can assume that the user types a valid URL that points to a well-formed and valid XML file which has the same format as the sample XML page. (Extra credit: add proper error checking to remove that assumption.) When he or she clicks submit, this album should be loaded and stored in the database. The person loading the album (obviously) becomes its owner. As with assignment 2, when you add a photo to the database you should first check if it already exists there so that duplicate entries are not created for the same photo.
